# Hybrid-under-sampling-and-oversampling-for-handing-imbalanced-credit-cared-data
> Credit card fraud detection is one of the most important applications of machine learning in the banking and financial sector. A major challenge in fraud detection systems is the presence of imbalanced data. In a real-world credit card dataset, the number of legitimate transactions is extremely high when compared to fraudulent transactions. For example, out of thousands or millions of transactions, only a very small percentage may actually be fraud cases. Because of this imbalance, machine learning algorithms tend to become biased toward the majority class, which is the normal transactions, and may fail to correctly identify fraud transactions. Even if a model achieves very high accuracy, it may still perform poorly in detecting fraud because it simply predicts most transactions as legitimate. Therefore, handling imbalanced data becomes an essential step in building an efficient fraud detection model.

> To solve the imbalance problem, sampling techniques such as oversampling and undersampling are widely used. Oversampling is a method in which the minority class, which represents fraudulent transactions, is increased in size so that the dataset becomes balanced. One simple oversampling method is Random Oversampling, where existing fraud records are duplicated multiple times. However, duplicating the same data repeatedly may cause overfitting because the model memorizes the repeated records instead of learning meaningful patterns. To overcome this issue, advanced oversampling techniques such as SMOTE (Synthetic Minority Oversampling Technique) are used. SMOTE generates new synthetic fraud samples by creating data points between existing minority class samples. Instead of copying fraud records, it produces new artificial samples that resemble real fraud transactions. This helps the model learn better fraud patterns and improves the overall fraud detection capability.

> Undersampling is another balancing technique used for handling imbalanced credit card datasets. In undersampling, the number of majority class samples, which are the legitimate transactions, is reduced to make the dataset balanced. Random Undersampling removes some normal transaction records randomly from the dataset. This technique reduces the size of the dataset and helps decrease training time. However, removing too many legitimate transaction records can result in loss of important information, which may affect the model’s performance. Advanced undersampling techniques such as NearMiss select only the most useful majority class samples that are closer to the minority class samples. This helps the model focus on difficult transaction patterns and improves classification performance.
> Hybrid undersampling and oversampling is a combined approach that uses both techniques together to balance the dataset more effectively. In this method, undersampling is first applied to reduce the excessive majority class samples, and then oversampling techniques such as SMOTE are used to increase the minority class samples. This combination helps overcome the disadvantages of using only one method. If only oversampling is used, the dataset may become too large and may introduce noise or overfitting. If only undersampling is used, valuable information from legitimate transactions may be lost. Hybrid sampling balances the dataset while maintaining important information and reducing unnecessary data.
> Overall, hybrid undersampling and oversampling is considered one of the most effective approaches for handling imbalanced credit card datasets. It combines the strengths of both sampling techniques and reduces their individual limitations. By creating a balanced dataset, hybrid sampling improves fraud detection accuracy, enhances machine learning performance, reduces bias toward majority classes, and helps financial institutions build secure and reliable fraud detection systems.
